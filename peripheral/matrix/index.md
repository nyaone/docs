# 喵窝 Matrix 注册与使用

喵窝部署有配套的 Matrix 服务。以下是详细的注册及使用教程，方便您参考。

## 账号关联

我们使用的是基于 OIDC 进行身份认证的方案，不支持传统的账号密码登录。您的 Matrix 身份与您的 Misskey 账号唯一关联，使用相同但均为小写的用户名作为您的 Matrix 身份标识。

有部分客户端可能不支持 OIDC 模式的身份认证登录，我们推荐您使用一些较为成熟的客户端解决方案。如您真的很希望使用这些客户端，您可以尝试与开发者取得联系，与其分享详细的 OIDC 身份验证流程。

## 使用

出于对用户信息的保护政策， Matrix 具有严格的端到端加密验证要求，您新登录的设备需要在旧设备上完成验证才可解密旧消息；您需要备份安全密钥，并妥善保管，以便保证其他设备上的会话可以被正常解密。当您在新设备登录时，您需要使用设备交叉验证，或是使用安全密钥来恢复先前收发的消息。

若您登出了旧设备，您从该设备上发送的消息会被标记为“由已删除的会话加密”。

我们推荐您使用 element 作为连接使用的客户端，您可以在 [im.nya.one] 使用已经预先配置好的网页版 element ，或是在 [element.io/get-started] 下载客户端。当然，您也可以使用任何支持 Matrix 协议的客户端，您可以在 [matrix.org/clients] 查看到官方认可的应用列表。

我们的 Matrix 服务器已经支持扫描二维码快速登录，如果您使用的是兼容的客户端（如 Element X ），您可以使用这个功能快速地在不同设备上登录您的帐号。如果您使用的是本地客户端登录的方案，你需要手动配置 <ruby>家服务器<rt>homeserver</rt></ruby> 域名为 `nya.one` 。通常来说客户端会自动拉取这个域名对应的实际服务器地址，但如果您使用的客户端没法成功获取，您可能需要手动配置服务器的地址为 `https://matrix.nya.one` 。

您会默认加入到我们实例的公开群组 `#public:nya.one` ，欢迎各位加入，也欢迎您加入其他任何的其他实例群组。玩得愉快！
